# Product Context

## Problem
В Go-экосистеме нет простого, встраиваемого S3-подобного key-value хранилища, которое:
1. Работает в single-process без настройки (просто `New(path, bucket)`)
2. Поддерживает multi-process без переключения API (прозрачный failover через Unix socket)
3. Даёт метаданные объектов (content-type, checksum, timestamps) — как S3
4. Опционально работает как HTTP-сервер

## Solution
S3Lite реализует всё это одной библиотекой без внешних зависимостей (кроме BadgerDB):

### Режимы использования
- **Single-process** — `s3lite.New()` → прямой BadgerDB в той же программе
- **Multi-process** — `multy.New()` → первый процесс открывает Badger и становится сервером, остальные подключаются через Unix socket. При смерти сервера клиент автоматически перехватывает lock и становится новым сервером.
- **HTTP сервер** — `serve.New()` → S3Lite работает как HTTP-сервер по протоколу S3 (можно использовать S3-совместимые клиенты)

### Ключевые сценарии
1. **AI assistant контекст** (исходная мотивация multy) — несколько AI-агентов на одной машине используют общую KV-базу для контекста, моделей, диалогов
2. **Embedded storage** — серверному приложению нужно что-то проще SQL, с S3-like API
3. **S3-совместимое хранилище** — для разработки и тестов на localhost
